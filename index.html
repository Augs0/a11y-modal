Accessibility Resources

There are a number of resources already available to help you.

  • Udacity course by Google: a basic course on how to improve your code for use with screen readers
  • Aria Guidelines: general guidelines on ARIA, including when and where to use certain roles
  • Colorable: check how readable your font and background colour choices are.
  • Inclusive Component Design: see ARIA in use in examples. This site has a lot of handy info about how to get around common issues for common components.
  • 'Web Accessibility' extension for VS Code: provides warnings in the terminal whenever accessibility adjustemnts are recommended.
  • 'Nu' HTML validator: validates HTML and checks for basic accessibility features
